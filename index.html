Proposed System

This System will perform the entire process of capturing the details of both the Mother and Child and store them securely in the hospital's database.
Most hospital appointments for checkups would be forgotten by the Mother, but this system notifies the Mother when the date is almost near via an Email and a text .
It also keep a record of the infant's progress and a register of the vaccine shots the Child is supposed to receive in nine months.

Objectives

1) To develop a module that will register a Mother with her Childs details to the maternal database system.


2) To develop a module that will keep track of the child's health progress for every scheduled appointment.

3) To develop a module that can help government clerk in conducting statistics of the new born babies after certain period of time.

Implementation

It will be achieved by the use of PHP scripting which will be used to code at the backend to develop a connection between the interface and the database.

JAVASCRIPT, Html and CSS would be in cooperation properly to develop an interactive and productive User Interface (U.I) that the users will be interacting with .
Wamp server will be applied to create and manipulate the database where information will be stored and accessed on demand

Methodology

The study will adopted Iterative Methodology, which takes the incremental model and cycles it several times in small increments (iterations).
Rather than stretch the entire project across the phases of the software development, each step is turned into several mini-projects that can add value as the product evolves.
The model will integrate the steps incrementally to form a complete software development methodology
